📚 Lexique rsync
OptionSensConseil
-aArchive : récursif + dates + liens + permissions + owner/group.Base d’un backup. Avec SMB/NFS, garde souvent --no-owner --no-group --no-perms.
/remotes/Sécurité cible réseau : le script refuse d’écrire sous /remotes si aucun montage actif n’est détecté.Évite de remplir l’image Docker quand un partage NFS n’est pas monté.
--deleteSupprime côté cible ce qui n’existe plus côté source.À tester avec --dry-run avant une première vraie passe.
--exclude=PATTERNIgnore un fichier ou dossier de la source.Exemple : AMELYS/, *.tmp, .Recycle.Bin/.
--delete-excludedSupprime aussi côté cible ce qui est exclu.Dangereux : décoché par défaut. Sans cette option, les exclus sont protégés côté cible.
--dry-runPré-analyse : calcule d’abord le volume à copier sans modifier, puis lance la vraie copie.Permet d’afficher une progression réelle du type 54 Go copiés / 4 To.
-WWhole file : recopie le fichier entier, pas l’algo delta.Souvent plus simple/rapide en local ou LAN rapide.
--partialGarde les fichiers partiels en cas de coupure.Utile pour gros fichiers.
-cChecksum : compare le contenu réel.Très sûr mais très lent.
-zCompression pendant transfert.Utile en SSH/VPN lent, inutile localement pour vidéos.
--numeric-idsGarde UID/GID numériques.Utile en Docker/Unraid pour éviter les noms utilisateur.
-xUn seul filesystem.Empêche de traverser d’autres montages.
--inplaceÉcrit directement dans le fichier cible.À éviter sauf besoin précis.
--statsRésumé final.Très utile dans les logs.
ℹ️ Principe
Ce module Yoleo fonctionne en host : Flask génère des scripts Python indépendants qui appellent rsync. Le tableau ne contient pas la logique de backup ; il lit seulement les métadonnées du script, lance le script dans une session tmux autonome via systemd-run quand disponible, et affiche son log.
SCRIPTS_DIR={{ conf.SCRIPTS_DIR }}
LOG_DIR={{ conf.LOG_DIR }}
STATUS_DIR={{ conf.STATUS_DIR }}
RSYNC_BIN={{ conf.RSYNC_BIN }}
PYTHON_BIN={{ conf.PYTHON_BIN }}
EXECUTION=systemd-run + tmux
🧱 Exemple Task Manager
Dans le gestionnaire de tâches, vous pouvez lancer un backup avec une commande du genre :
python3 {{ conf.SCRIPTS_DIR }}/backup_nextcloud.py
